About Me

Education and Interests

My interest in computer science first started in junior high, when I enrolled for a game development class. We started with a brief introduction to C++, but soon switched to using a game engine. I was fascinated by the power of code and the creativity it enabled, which ultimately inspired me to study CS. This remains one of my favorite classes of all time.

From 2021-2025, I completed my undergraduate degree at San Diego State University. At SDSU, I studied a wide range of topics, including Algorithms & Data Structures, Computer Networks, Computer Security, Cloud Computing, UNIX System Administation, Database Theory and Implementation, Machine Learning, and Data Science, among others. During my undergraduate program, I placed a strong emphasis on my academic performance. While consistently working 20 hours a week and leading a student organization at SDSU, I maintained a 3.93 GPA in major-related courses. In May of 2025, I obtained my Bachelor's of Science, graduating Summa Cum Laude. I was also identified as one of the 2025 Outstanding Computer Science Undergraduate Student by the department.

This fall, I'll begin pursuing my Master's of Science at UC Davis. One of the things that I love about computer science its breadth. Computer Science is a diverse field spanning topics, and if I could, I'd study them all. However, while pursuing my master's, my topics of focus will be Software Engineering, Programming Languages, and Computer Science Education.

Personal Projects

A more complete list of my projects can be found on my website.

RollCall: A Cloud-based Virtual Guestbook App.

One of my favorite personal projects that I've worked on is RollCall, a full-stack, cloud-based web app that allows users to create, manage, and share digital guestbooks for events. The idea was to allow guests to sign these digital guestbooks, giving hosts the opportunity to build hype before an event and preserve a digital momento after it ended. The key features we aimed to implement were:

  1. Support for CRUD (Create, Read, Update, Delete) operations on digital guestbooks.
  2. Implementation of a robust user-authentication mechanism to ensure guestbooks are secure.
  3. Message moderation capabilities for guestbook hosts which are not visible to guests.

I contributed to this project at many stages throughout the project lifecycle from system design to deployment. During the design phase, I helped draft the user-flow diagram and overall system architecture diagram. I also led discussions regarding our tech stack, which ultimately consisted of Python (Flask), HTML/CSS/JavaScript, SQL, Git/GitHub, Docker, Bash, and Google Cloud Run.

Later, I implemented the SQL schema, deploying it to a Google Cloud SQL instance, and configuring access via a service account using Google Cloud IAM. To support serverless deployment, I created a Dockerfile for containerization, and contributed to a Bash deployment script using the Google Cloud CLI to push the container image and launch the service on Cloud Run.

With our database configured, I then developed Flask-SQLAlchemy ORM models to simplify database interactions, and contributed to the frontend Jinja2 templates for dynamic content rendering. I was also primarily responsible for implementing core features such as user authentication and message moderation.

F4E: A Custom Discord Bot.

Another personal project I enjoyed working on was F4E, a custom Discord bot built for a personal server with friends. This bot was programmed in Python, and version controlled with Git. The purpose of this bot was to respond to user text commands with jokes, riddles, trivia, etc., while also being capable of dynamically generating visual content such as custom memes.

To achieve this, I integrated multiple external APIs. One of my favorite commands leveraged the Art Institute of Chicago's API, allowing the bot to implement an "art exhibit of the day." I also regularly consulted the official documentation for the Discord API to handle things such as asynchronous message processingand user permissions correctly.

This project gave me hands-on experience with real-time API integration and asynchronous programming. There was also some text sanitization required to properly handle some commands, especially with regards to dynamically generated content.

VICE: A Chess Engine.

My first large-scale personal project was a chess engine made entirely in C. I've been playing chess for a few years, and at the time I began working on this project, the creation of a chess engine seemed like the perfect blend of interests.

I followed and implemented the VICE Chess Engine, which was based on Bluefever software’s video series. Written in C, this project introduced me to core concepts common to many board game engines and decision-based systems such as move generation, minimax search, and alpha-beta pruning. I later experimented with tuning evaluation scores and implementing an opening book to improve the engine's strength and search performance.

This project strengthed my understanding of low-level programming, as I had more practice with concepts such as memory management. It also gave me more practice with bit manipulation, since the use of a bitboard encoded the game state into an integer. To streamline the build process, I also configured a Makefile, enabling easier recompilation.
